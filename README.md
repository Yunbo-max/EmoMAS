# EmoMAS: Emotion-Aware Multi-Agent System for High-Stakes Edge-Deployable Negotiation with Bayesian Orchestration

> 🎉 **Accepted at ACL 2026 (Main Conference, AI/LLM Agents track).** Rated top 50% of accepted papers by the area chair &mdash; roughly the top 9% of the 12,148 submissions.

EmoMAS is a Bayesian multi-agent framework that transforms emotional
decision-making in negotiation from reactive to strategic. A Bayesian
orchestrator coordinates three specialised agents — **game-theoretic**,
**reinforcement-learning**, and **psychological-coherence** — fusing
their real-time predictions while continuously re-weighting agent
reliability from negotiation feedback. The result is online emotional
strategy optimisation **without any pre-training**, suitable for
privacy-sensitive, on-device settings where LLMs cannot be deployed.

We further release four high-stakes edge-deployable negotiation
benchmarks spanning debt, healthcare, emergency response and education.

## ✨ Contributions

1. **Bayesian Orchestrator** that performs multiplicative reliability
   updates over three specialist agents both at the macro (trajectory)
   and micro (per-turn) level.
2. **Online** emotional strategy learning — no pre-training, no fixed
   policy, adapts to each opponent within a single dialogue.
3. **Four new edge-deployable negotiation benchmarks** (debt,
   healthcare, disaster response, education) with 100 scenarios each.
4. Consistent gains of both SLMs (Qwen-7B) and LLMs (GPT-4o-mini) when
   equipped with EmoMAS, while reducing manipulation rate relative to
   single-agent baselines.

## 🚀 Quick start

```bash
git clone https://github.com/Yunbo-max/EmoMAS.git
cd EmoMAS
pip install -r requirements.txt

# Configure API keys
cp .env.example .env
# edit .env and set OPENAI_API_KEY=sk-...

# Smallest possible run to verify everything works
python experiments/run_all_datasets.py \
    --model_type bayesian --dataset_type debt \
    --scenarios 1 --iterations 1 --max_dialog_len 4
```

## 🧪 Running experiments

The canonical driver is `experiments/run_all_datasets.py`. Supported
`--model_type` values:

| Flag | What it runs | Paper reference |
|------|--------------|-----------------|
| `vanilla` | Negotiator without emotion control | Table 1 row "Vanilla" |
| `prompt` | Random-emotion prompt baseline | Table 1 row "Vanilla+Prompt" |
| `gametheory` | Game-theoretic agent alone | §3.2 |
| `rl_agents` | Tabular Q-learning agent alone | §3.3 |
| `coherence` | Psychological coherence agent alone | §3.4 |
| `gpt` | LLM orchestrator (EmoMAS-LLM) | Table 1 row "EmoMAS-LLM" |
| `bayesian` | Full Bayesian orchestrator (**EmoMAS-Bayes**) | §3.5 / Table 1 row "EmoMAS-Bayes" |

Supported `--dataset_type`: `debt`, `disaster`, `student`, `medical`, `all`.

Typical paper-reproduction run:

```bash
python experiments/run_all_datasets.py \
    --model_type bayesian \
    --dataset_type all \
    --iterations 5 --scenarios 100 \
    --model_creditor gpt-4o-mini --model_debtor gpt-4o-mini \
    --debtor_model_type vanilla
```

For small-model (SLM) runs, use `--model_creditor qwen-1.5b` or any
other HuggingFace model supported by `llm/llm_wrapper.py`.

## 📊 Datasets

All four CSV datasets live in `data/`. Each contains 100 fully
synthetic scenarios generated by gpt-5 for controlled experimentation:

| Dataset | File | Domain |
|--------|------|--------|
| CRAD | `credit_recovery_scenarios.csv` | Debt collection |
| SSD | `hospital_surgery_scenarios.csv` | Surgical scheduling |
| DESRD | `disaster_survivor_scenarios.csv` | Disaster rescue |
| SSAD | `education_sleep_scenarios.csv` | Student sleep alerting |

These are intended strictly for research. They are **not** representative of real
high-stakes human interaction and must not be deployed in any operational
setting without human oversight and domain-specific safety review.
